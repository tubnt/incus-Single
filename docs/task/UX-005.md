# UX-005 用户创建云主机入口重做：独立整页 `/launch`

- **status**: completed
- **completedAt**: 2026-05-02
- **priority**: P2
- **owner**: claude
- **createdAt**: 2026-05-02
- **scope**: 用户决策 "proceed 1"：把"用户创建云主机"从 `/billing` + 抽屉重设计为独立整页 `/launch`，与管理员 `/admin/create-vm` 结构对齐

## 现状（UX-004 之后仍然存在的根因）

1. 入口在 `/billing`（账单中心），账务噪声压制创建动作
2. PurchaseSheet 32rem 抽屉 → 桌面端只占 22%，无法承载完整 IaaS 创建流
3. "选规格"在抽屉外、"配 OS / 名称"在抽屉内 → 流程被强行劈成两段
4. 入口路径：`/vms 顶栏 新建 VM` → `/billing` → 滚找产品卡 → 抽屉 = 3 次上下文切换

## 方案

### 1. 新增 `web/src/app/routes/launch.tsx`（~280 行）

线性整页，左主表单 + 右 sticky Summary：

```
左栏（lg:col-span-2，flex flex-col gap-8）
  ① 套餐 (Plan)        — 卡片网格，大字 vCPU/RAM/SSD + 月价，active 态强化
  ② 系统镜像 (OS)       — 复用 UX-004 OsImagePicker (Combobox)
  ③ 认证 (Authentication) — SSH Key 提示（已绑/未绑双态），OS 镜像默认会注入
  ④ 主机名 (Hostname)    — Input + 实时校验 + 自动生成 placeholder
右栏（lg:col-span-1，sticky）
  · 你的选择 4 行 SummaryRow
  · 月价大字
  · 当前余额 + 不足提示
  · [立即创建] 主 CTA
mobile：MobileBottomBar 主 CTA
```

provisioning / 凭据揭示沿用 PurchaseSheet 的 SSE + SecretReveal 模式，但落在同一整页内（不再开抽屉）：FORM → PROVISIONING(JobProgress) → DONE(凭据网格) | FAILED。

### 2. 改入口 `/vms` → `/launch`（4 处）

- PageHeader actions 主 CTA 链接
- EmptyState 主 CTA 链接
- 键盘 `n` 快捷键 navigate target
- useCommandActions `user.create-vm` perform target

### 3. `/billing` 退化为纯账户中心

- 删除 "可选套餐" section（产品卡 + ProductCard 子组件）
- 顶部 BalanceCard 旁加 "+ 新建云主机" 链接到 `/launch`
- 移除 PurchaseSheet 导入与使用
- 保留 OrderRow（已有 pending order 还可走 `pay` 按钮 → 同步路径凭据用 `setCredentials` 仍展示在 billing 顶部）

### 4. 删除 `web/src/features/billing/purchase-sheet.tsx`（~340 行）

完全删除。

### 5. i18n 新键

- `nav.launch` "创建云主机" / "Launch VM"
- `launch.title` `launch.description` `launch.planTitle` `launch.planHint`
- `launch.authTitle` `launch.authHint` `launch.hostnameTitle` `launch.hostnameHint`
- `launch.cta` `launch.creating` `launch.balanceWarn` `launch.summaryYourPick`
- `launch.provisioning` `launch.provisioningHint` `launch.done` `launch.failed`
- `launch.balanceLabel` `launch.balanceInsufficient`
