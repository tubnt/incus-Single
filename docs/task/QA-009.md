## 2. P1 详情

### N-01 SSE 无最大重连次数 / 无超时熔断
- **位置**：`web/src/shared/lib/sse-client.ts:48-101`、`web/src/features/jobs/use-job-stream.ts:78-81`
- **代码**：
  ```ts
  // sse-client.ts: while (!stopped) { ... if (opts.onError?.(e) === false) return; ...
  //   await new Promise((res) => setTimeout(res, reconnectMs)); }
  // use-job-stream.ts:
  onError: () => { connectedRef.current = false; /* 默认行为：返回 undefined → 自动重连 */ },
  ```
- **复现**：触发任意异步 job（VM 创建 / 重装 / 批量迁移 / 节点加入），中途 `pkill -9 incus-admin` 让后端断流。前端 SSE 无限 3s 间隔重连，UI 永远不会进入 terminal 态，关闭抽屉重开 `terminal` 仍 `null`。
- **修复**：在 `useJobStream.onError` 累计 N 次（如 5 次）或累计 60s 后 `dispatch({type:"done", status:"failed"})`，并 toast `连接中断，请刷新页面查看最终状态`。

### N-02 /admin notFoundComponent 硬编码英文
- **位置**：`web/src/app/routes/admin.tsx:31,36`
- **代码**：
  ```tsx
  <p className="text-muted-foreground">Admin page not found</p>
  ...
  <a href="/admin/clusters" className="...">Back to Clusters</a>
  ```
- **复现**：zh 用户访问 `/admin/notexist` → 全英。
- **修复**：包 `t("admin.notFound.title")` / `t("admin.notFound.cta")`，zh / en common.json 各加一对 key；`href="/admin/clusters"` 与 `admin.tsx:24` 默认跳转 `/admin/monitoring` 不一致，统一一边。

### N-03 路由 errorComponent 缺失
- **位置**：`web/src/app/routes/__root.tsx:15-18` + 全部子路由
- **代码**：
  ```ts
  export const Route = createRootRoute({ component: RootLayout, notFoundComponent: NotFound });
  ```
- **复现**：任意 route loader（如未来加 loader 的页面）抛出 → TanStack Router 默认渲染最近 errorComponent；当前 root 没设，子路由也没设 → 控制台报错，UI 白屏 / 仅剩侧栏。
- **修复**：root 加 `errorComponent: ErrorBoundaryFallback`（已有 `ErrorBoundary` 组件可复用一个 RouterError 变体），fallback 带 t() 文案 + "重试" 按钮。

### N-09 firewall 规则前端无格式校验
- **位置**：`app/routes/admin/firewall.tsx:276-296`
- **代码**：destination_port / source_cidr / description 都是 `<Input type="text">`，onChange 直传 patch
- **复现**：输入 "abc-def" 端口范围 / "300.0.0.0/8" CIDR → 直接 mutate → 后端 400 / 500
- **修复**：添加 `<input pattern>` 或 onBlur 校验 + 错误标红 + 禁 disabled save。port-range RE：`^(\d{1,5})(-\d{1,5})?(,\s*\d{1,5}(-\d{1,5})?)*$`；CIDR：`^(\d{1,3}\.){3}\d{1,3}/\d{1,2}$` + 范围检查。

### N-10 充值 customAmount 上限缺
- **位置**：`features/users/components/user-detail-sheet.tsx:182-205`
- **代码**：`<Input type="number" min="0" ... />`，无 `max`
- **复现**：管理员对单个用户充 `999999999.99` 或科学计数 `1e10`，前端无阻止
- **修复**：写 `max={999999}`（或与产品商定的单次上限）+ `customExceeds` 派生态在 `> max` 时禁 button + 红字提示。

### N-11 Stateful 启用失败后 banner 不重置
- **位置**：`features/vms/components/migrate-batch-sheet.tsx:114-143`
- **现象**：mutate `enableStatefulBatch` 失败 → toast.error 给出 `.message`，但 `nonStateful` 数组没刷（因为是 `selectedVMs` 的派生），用户再看到同样 banner、同样按钮，不知"失败"还是"未启用"。
- **修复**：mutation 加 onError 状态外露 + banner 上加 retry/error 红色提示；或 onError 时主动 invalidate `vmKeys` 触发派生态刷新。

### N-12 /tickets/<id> 邮件深链不可达
- **位置**：`app/routes/tickets.tsx`
- **现象**：当前 route 只有 `/tickets`（list + 内嵌 detail by query `?id=`），`/tickets/123` 落到 root NotFound（友好但失去上下文）
- **修复**：要么在邮件中始终用 `?id=` 查询参数；要么加 `tickets.$id.tsx` route 文件 + 同样 detail 渲染。

### N-13 节点加入向导回退状态污染
- **位置**：`app/routes/admin/node-join.tsx:146-202`（reducer + dispatch back）
- **现象**：从 `confirm` 步骤返回 `cred` 修改 host，再前进，旧 `fingerprint` 仍在 state（reducer 只改 stage 不清子状态）
- **修复**：`back` action 在跨 stage 回退时同时清掉关联子状态：cred → 清 fingerprint+confirm；fingerprint → 清 confirm。

### N-14 firewall 默认组全删后无前端 guard
- **位置**：`features/firewall/components/default-groups-manager.tsx:109-116`
- **现象**：用户可全删默认组，新建 VM 时若后端兜底缺失，VM 不绑任何组（裸奔）
- **修复**：UI 删最后一条时弹强提示 + typeToConfirm；或后端兜底 `ensureDefault()` 已存在则不动。

### N-15 Floating IP RELEASE 二次确认强度不够
- **位置**：`app/routes/admin/floating-ips.tsx:74-87`

- **修复**：N>1 时强制让用户输入"释放 N 个 IP" 这种带数量的字符串；N=1 时输入 IP 自身。

### N-16 useJobStream.onError 永远重连
- **位置**：`features/jobs/use-job-stream.ts:78-81`
- **代码**：

### N-17 AI 诊断面板 XSS 前瞻
- **位置**：`features/jobs/components/ai-diagnose-panel.tsx:78-102`
- **状态**：当前用 `{value}` JSX 文本渲染，React 自动 escape，**无现存 XSS**
- **风险**：若未来需要渲染 markdown / HTML / 链接，常见误区是直接换 `dangerouslySetInnerHTML`（LLM 输出本身就可能含 `<script>`、`<img onerror>` 等 payload）
- **修复**：在文件顶部加注释钉死"Never use dangerouslySetInnerHTML for AI output. Use markdown-to-jsx with explicit allowList."，并加测试用例。

### N-18 favicon dev 404

- **检查**：`web/public/` 存在 `favicon.svg`（`index.html` 引用），但浏览器旧约定还是去找 `/favicon.ico`
- **修复**：`public/` 加一个 16x16 ico fallback；或在 index.html 显式 `<link rel="shortcut icon" href="/favicon.svg">`（已有 type="image/svg+xml"，部分浏览器仍 fallback）。

### N-19 SSH key 名缺 maxLength

- **修复**：`maxLength={64}` + 文案。

### N-20 firewall warning 模板硬编码

- **代码**：`toast.warning(\`\${res.warning}: \${res.sync_err ?? ""}\`)`

### N-21 批量迁移缺总耗时

- **修复**：SheetDescription 后追加 `预计 {{n*30}}–{{n*120}} 秒（{{n}} 台 × 30s-2min）`。
