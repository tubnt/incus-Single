### Bug #1 — `volatile.*` config 被 PUT 回 Incus 触发 forbidden (P1)
- 触发：`reset-password mode=offline` 第一次提交 → 500 `Setting "volatile.eth0.host_name" on virtual-machine ... is forbidden`
- 影响：所有 GET-modify-PUT 模式（`resetPasswordOffline` / `floating_ip.attach/detach` / `firewall.bind/unbind`）都有隐患；vm-a3e86d 测试时偶然没触发是**运气**

- 单测：`TestStripVolatileConfig` 5 case
- ✅ 重测 reset-password offline 200 channel=offline，floating IP attach 200 + 整闭环 OK

### Bug #3 — Migrate 不带 `live: true` (P1)
- 触发：UI 点 Migrate node1→node2 → 500 `Instance must be stopped to be moved statelessly`
- 根因：handler `MigrateVM` body 只发 `{name, migration:true}`，未启用 live 标志
- 修复：改为 cold migrate 路径 — 自动 stop → migrate → start（VM 创建时未设 `migration.stateful=true`，true live migration 走不通）；增 `vmIsRunning` helper 判断是否需要 stop
- ✅ Incus CLI 直接验证 cold migrate 闭环 OK

### Bug #5 — VM 创建时未设 `migration.stateful: true` (P2)
- 影响：即使后端发 `live:true` Incus 也拒绝（"Live migration requires migration.stateful to be set to true"）
- 修复方向：要么 VM create 时设 `migration.stateful: true`（影响 cold-storage VM 性能），要么走 cold migrate（已选）
- 当前选 cold migrate 兼容现有 VM；create 时设 `migration.stateful` 留作可选 P3 增强

### Bug #8 — Reinstall delete 不等待 async 完成 → recreate "already exists" (P1)
- 触发：UI Reinstall 提交 → 500 `recreate instance: incus error: Instance "vm-8f8912" already exists`
- 根因：`service.Reinstall` 调 `client.APIDelete` 但忽略返回的 async operation，立即调 APIPost 创建同名 → Incus 还在删除中
- 副作用：VM 在 Incus 已删但 DB 还 running（reconciler 后续 60s 标 gone）→ **VM 永久消失，用户数据丢失**
- 修复：捕获 delete async response → `WaitForOperation` → 然后才 create；同时 strip volatile config 给 create body
- ⚠️ **未补救机制**：如果 create 失败（如镜像不存在），VM 已删除无法回滚。建议加 dry-run pre-check：先 `images:` 验证目标 source 可拉取，再 delete
- ✅ 修复已部署 dist_hash=待重测

### Bug #14 — 所有 API 错误信息仅显示 `HTTP 400:` (P1) 🟡 本地已修，未部署
- 触发：SSH Keys 添加 `not-a-valid-key` → toast 只显 `"HTTP 400:"` 没有任何有用信息；同 Bug #9 根因
- 根因：`shared/lib/http.ts` `HttpError` 构造 `message = HTTP ${status}: ${statusText}`，抛出后前端 `toast.error((err as Error).message)` 只拿到 statusText（对 HTTP/2 常为空）；真正 body.error 被扔到 `.body` 但没人看

- 修复：`HttpError.message` 用新 helper `formatHttpErrorBody(status, statusText, body)` —— 按顺序回退 `body.error` → `body.error + (body.details)` → `body.message` → body string → statusText → status code
- 生产复现：ai@5ok.co /ssh-keys 提交 `name=test-key publickey=not-a-valid-key` → 原来只 `HTTP 400:`；修复后应显示 `HTTP 400: invalid SSH public key`
