## 修复

`internal/service/vm.go::Reinstall` 在 `client.APIDelete` 之前新增 `prePullImage`：

1. POST `/1.0/images?project=<proj>` 提交镜像源 spec，触发 Incus 拉取到本地缓存
2. WaitForOperation 等异步完成（5min timeout）
3. 拉失败 → return early，**原 VM 不动** —— 把"上游可达但 alias 不存在"的失败提前到删除前
4. 拉成功 → 即使后续 probe → recreate 之间上游挂掉，create 也能命中本地缓存

`prePullImage` 用 `incus image copy mode=pull` 等价语义，幂等：镜像已在 cache 时 Incus 短路返回。
