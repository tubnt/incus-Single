## Description

PLAN-026/027 后续 3 个用户决策项：

- **B2 admin batch create**：`POST /admin/clusters/{name}/vms` 接 `count: N` 字段（1..16），一次入队 N 个 jobs；返回 `job_ids[]` + `vm_ids[]`。前端 form 加数量输入 + toast 列出所有 job_id（用户走 /admin/jobs 看进度）。
- **C2 cluster-env.sh 生成**：`GET /admin/clusters/{name}/env-script` step-up auth + admin only，从 `incus cluster members` + DB 派生 cluster-env.sh 内容（`CLUSTER_NODES` 数组按 5ok.co 网络拓扑约定从 mgmt IP 反推）。返回 `Content-Type: text/x-shellscript`。
- **D2 maintenance mode**：`POST /admin/clusters/{name}/nodes/{node}/maintenance` 接 `{enabled: bool}`，调 Incus `PATCH /cluster/members/{name}` 设 `scheduler.instance=manual`。enabled=false 还原为 `all`。前端 NodeDetail 卡加 toggle，与 evacuate/restore 平级。

## Acceptance criteria

- B2：admin POST `count: 3` → 返 202 + `job_ids: [N1, N2, N3]`。前端 form 1..16 输入 + toast 列出
- C2：admin GET env-script 路由 → 必须 step-up 通过；返回 cluster-env.sh 文本（`CLUSTER_NODES` 数组内容来自实际 cluster + ops fill TODO）；非 admin / 未 step-up 返 401/403
- D2：admin POST maintenance enabled=true → Incus member.config.scheduler.instance=manual；前端 toggle 显示当前状态
- 后端 `go vet` clean / `go test` 全绿；前端 tsc 0 / vitest 全绿
- 生产 vmc.5ok.co 部署 + systemd active

## Notes

`cluster-env.sh` 生成 V1 假设 5ok.co 网络拓扑：mgmt 10.0.10.X / ceph_pub 10.0.20.X / ceph_cluster 10.0.30.X 都用 pub IP 末位。其他网络拓扑需 ops 手工调整或 customize 模板字段。
