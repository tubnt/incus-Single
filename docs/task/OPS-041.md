## 描述

当前 PLAN-033 已落地 probe + heuristics，但 `computeHeuristics` 仅按硬编码 CIDR (`10.0.10/20/30.x`) 匹配，对**未预配 IP 的新节点**或**异构硬件 (NIC 数量/速率/命名不一致)** 失效。本任务用"模板 + AI"分层架构补齐：

1. **Tier 1 — 确定性 ranked 候选**（probe 增强 + heuristic v2，覆盖 80% 标准节点）
2. **Tier 2 — AI 增强角色推荐**（仅在 Tier 1 置信度低 / 用户主动触发时调用，输出强制 JSON schema）
3. **Tier 3 — 失败诊断**（join-node.sh 失败时把 stderr + 上下文喂 LLM，输出分类 + 修复建议）

**核心安全约束**：

- AI 永不直接执行命令（仅生成结构化建议；执行体仍是模板 / join-node.sh）
- 探测数据脱敏后才送出（MAC 哈希、IP 截断到 /24）
- AI 输出强制 JSON Schema 校验（失败则回退 Tier 1）
- env `AI_PROVIDER` 可关闭整条 AI 路径
- 限流 + 结果缓存 + 全量 audit

不做：反向 agent / IPMI 自动 enlist（PLAN-033 已否决）；live migration；AI 自动重试修复。
