# OPS-011 日志降噪：用户取消请求不再打 ERROR

- **status**: completed
- **priority**: P3
- **owner**: claude
- **createdAt**: 2026-04-28 18:50
- **completedAt**: 2026-04-28 18:51

## 现象

生产 journalctl 周期性出现 ERROR：

```
{"level":"ERROR","msg":"user lookup failed","email":"ai@5ok.co","error":"find user: context canceled"}
```

`context canceled` 表示客户端主动断开（关浏览器/导航离开/超时）。这不是真错误，但被记成 ERROR 干扰运维监控视野，告警噪音里淹没真错误。

## 修复

`internal/middleware/auth.go::userLookup` err 分支加 `errors.Is` 判断：

```go
if errors.Is(err, context.Canceled) || errors.Is(err, context.DeadlineExceeded) {
    slog.Debug("user lookup aborted by client", "email", email, "error", err)
} else {
    slog.Error("user lookup failed", "email", email, "error", err)
}
```
