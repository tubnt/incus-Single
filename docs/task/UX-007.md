# UX-007 初始凭据可重看入口（Step-up gated）

- **status**: completed
- **priority**: P1
- **owner**: claude
- **createdAt**: 2026-05-10
- **completedAt**: 2026-05-10
- **scope**: 用户在 `/launch` 创建 VM 后，关闭页面 / 切走 / 网络断时
  DonePanel 密码"丢失"的 UX 缺口；后端实际能再读，前端没暴露入口。
  本任务给 portal `/vm-detail` 加"查看初始凭据"按钮 + step-up gate。

## 现状

- 后端：`vms.password` AES-256-GCM 落库；`GET /portal/jobs/{id}` 已经返
  password 给 owner（无 step-up，无窗口），但 portal UI 没人调
- 前端：DonePanel 仅创建瞬间显示一次；`/vm-detail` 详情页 GET
  `/portal/services/{id}` 走 `NewVMServiceDTO` **故意不带 password 字段**
- 用户体验：关页面 = 永久丢失

## 方案 A（已批 proceed）

后端：
- 新端点 `POST /portal/services/{id}/initial-credentials`
- step-up gated（middleware/stepup.go 加白名单）
- 校验 owner_id == userID；从 `vmRepo.GetByID` 拿解密后 password
- 返 `{ vm_name, ip, username, password, created_at }`
- 写 audit `vm.view_initial_credentials`
- 不引入新 DB 列；password 来源是已有 vms.password（OPS-022 加密）

前端：
- `app/routes/vm-detail.tsx`（portal）顶部加「查看初始凭据」按钮
- 点击 → mutation 触发 step-up redirect → 回来 replay → 弹 SecretReveal Sheet（60s 自动遮蔽）
- 与「重置密码」并存：tooltip 说明区别（前者读历史，后者生成新）
- DonePanel 加「下载凭据 .txt」一次性下载（一并补，零 step-up）+ 提示文案
- i18n zh + en，新增 6 个 key

视觉：DESIGN.md token 优先（status-info border / radius-card / size-sheet-sm）

## 风险

| 风险 | 缓解 |
|---|---|
| owner 高频查密码 | step-up 5min freshness gate + audit 全记 |
| 与"重置密码"语义混淆 | 两按钮独立 + tooltip + 分组（凭据 / 操作） |
| 解密失败（密钥轮换坏 / 旧明文行）| 返 422 + 提示"无法读取，请使用重置密码" |
| 旧 VM 未加密 | 走 decryptOnRead 兜底（passthrough，OPS-022 v1 已支持）|
