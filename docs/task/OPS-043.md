## 描述

启用 Incus live migration（不停机迁移），让 PLAN-037 批量迁移与 admin/vm-detail 单台迁移可选 live / cold。

前置条件已确认（vmc.5ok.co 5 节点）：
- ✅ Incus 6.23 支持 live migration
- ✅ ceph-pool 共享存储（driver=ceph）
- ✅ cluster network 10.0.20.0/24 互通
- ❌ 现存 4 台 VM 未设 `migration.stateful=true`（vm_create.go OPS-008 #5 起新建默认开，但旧 VM 没回填）

实现分 3 段：

1. **Stateful 回填**：
   - `POST /admin/vms/{name}/enable-stateful`：set config + restart VM（admin step-up gated）
   - `POST /admin/vms:enable-stateful-batch`：批量异步 job
   - admin/vms 列表加"待启用 live"列 + 一键开启
2. **VMService.Migrate 加 mode**：
   - `Migrate(ctx, ..., mode "auto" | "live" | "cold")`，默认 `auto`：probe stateful → live；否则 cold
   - live 失败自动 fallback cold + audit `vm.migrate.live_fallback`
3. **UI**：
   - VM detail / migrate-batch sheet 加 "Live migration（不停机）" toggle
   - sheet 顶 banner：选中集合中有 N 台不支持 live → 链接到批量回填

不在范围：
- QEMU 重编（无需，6.23 ubuntu 包已带 live patch）
- 旧 storage 迁移到 ceph（已就绪）
