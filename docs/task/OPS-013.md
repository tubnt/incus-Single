# OPS-013 admin/create-vm 页 UX 修复（i18n + 规格按钮 active 态）

## 现象

OPS-008 第 3 轮 UI 回归发现两处体验问题：
1. `/admin/create-vm` 标题是 "Create VM"，未走 zh i18n（zh 用户看到中英混排）
2. 4 个规格按钮（Small/Medium/Large/XLarge）**看不出哪个被选中**：active 样式只是 `bg-primary/10` 弱底色 + 同色边框，对比度低

附带：右下 Summary 和 VM 创建成功凭据卡片也是硬编码英文 `Cluster:`、`Config:`、`Name:`、`IP:` 等。

## 修复

### `web/public/locales/{zh,en}/common.json`

补 `admin.createVmTitle` / `admin.creatingVm` / `admin.vmCreated` / `admin.savePwdHint` / `admin.goToAllVms` / `admin.ipAuto` 双语 + `common.summary` / `common.failed`。

### `web/src/app/routes/admin/create-vm.tsx`

规格按钮 active 态强化：
- `border-2`（从 `border` 加粗）
- `bg-primary/15` + `ring-2 ring-primary/40 shadow-sm`
- 右上角加 `✓` 选中标记（`absolute top-1 right-1.5`）
- 选中时数字行也变 `text-primary/80`
- 加 `aria-pressed={active}` + `data-testid="spec-preset-{label}"` 给后续 E2E 用

Summary + 凭据卡片所有硬编码 label 改走 i18n（`vm.cluster` / `vm.config` / `vm.osImage` / `vm.project` / `vm.ip` / `vm.name` / `vm.username` / `vm.password` 已存在）。
