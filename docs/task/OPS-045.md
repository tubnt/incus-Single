# OPS-045 OS 模板目录补充 Windows（占位 + 默认用户映射）

## 描述

`os_templates` 目录目前只有 9 条 Linux 发行版（migration 009 seed），用户在 admin/os-templates 看不到 Windows 入口，且若管理员手动新建一条 Windows 模板，前端 OS 选择器会把它归到 "Other" 分组、`defaultUserForImage()` 会回落到 `ubuntu`。

linuxcontainers.org 不托管 Windows 镜像，无法像 `images:ubuntu/24.04/cloud` 那样直接拉。Windows VM 的运维路径是：管理员先把 ISO 转 qcow2 → `incus image import ... --alias windows-server-2022` 上传到本地 Incus，然后在 admin/os-templates 把模板 source 指向该 alias、protocol 改为 `incus`、清空 server_url、再启用。

本任务把这个流程的"占位骨架"提前补上，让"添加 Windows"成为一次后台配置而非代码改动。

## 改动范围

1. `db/migrations/023_windows_os_template.sql` — seed `windows-server-2022` / `windows-11` 两条 `enabled=false` 占位（source/protocol/default_user 已就位，等管理员补 server_url 或换 source 后启用）。
2. `web/src/features/vms/os-image-picker.tsx` — `FAMILY_DISPLAY` 加 `windows: "Windows"`；`FAMILY_ORDER` 把 Windows 排在 Other 之前；`familyFor()` 单独识别 `windows*` alias（无 `/version/cloud` 形式）。
3. `web/src/features/vms/default-user.ts` — 新增 `(^|[/:_-])windows` → `Administrator` 规则，放在 ubuntu 兜底之前。
4. `internal/handler/portal/reinstall_resolve.go::defaultUserForSource` — `windows` 分支回 `Administrator`，覆盖 legacy os_image 路径。
5. 单测：`reinstall_resolve_test.go` + `default-user.test.ts` 各加两条 Windows case。

## 不做

- 不改 `BuildCloudInit`：当前生成的是标准 `#cloud-config password/chpasswd/ssh_pwauth`，cloudbase-init 在 Windows 上会消费 `password` 字段（够用）。完整的 Windows unattended.xml / WinRM 配置不在本任务范围。
- 不预置真实 Windows 镜像 / ISO：合规与版权。管理员需自行上传到本地 Incus。
- 不在 admin/os-templates 表单加 Windows 专属字段（本任务保持表单不变）。

## 验收

- migration 023 在已部署 9 条种子的 DB 上 `ON CONFLICT DO NOTHING`，幂等；新部署直接两条占位入库。
- admin/os-templates 列表能看到两条 Windows 模板（停用态），点编辑可改 source/protocol/server_url 后启用。
- OS 选择器在 Windows 启用后显示 "Windows" 分组；选中 Windows 镜像时 `defaultUserForImage()` 返回 `Administrator`。
- `go test ./...` 全绿；`vitest` 全绿；`tsc --noEmit` / `bun run lint` / `bun run build` 全绿。

## 笔记

source 字段用单段 alias `windows-server-2022` 而非 `windows/server/2022/cloud`，是为了贴合 `incus image import --alias <name>` 的真实命名习惯；protocol 从默认的 `simplestreams` 改为 `incus`，触达本地 Incus image store 而不是远端 simplestreams。管理员若想把镜像挂到自托管 simplestreams 服务器，只需把 protocol 改回 `simplestreams`、把 server_url 填上。
