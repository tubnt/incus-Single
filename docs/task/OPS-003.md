## 描述

原密码重置路径只走 Incus guest-agent exec chpasswd，VM 关机或 agent 挂掉就没辙。Phase C 加"离线回落"：auto 模式先试 online，失败自动改写 cloud-init + 重启走 offline。

## 交付

- `service.ResetPassword` 签名改 `+mode`（auto / online / offline）
- offline 路径：stop → `cloud-init.vendor-data` 注 chpasswd.users[] + bump `cloud-init.instance-id` → PUT 全量 instance → start；不覆盖 user-data 保 SSH keys
- handler portal + admin 双写 `mode` 入参 + audit `channel/fallback`
- 响应体新增 `channel / fallback` 让前端显示走的路径
- 单测 3 组（YAML 形状 + instance-id 唯一性 + mode 枚举 wire 字符串）
- 生产合约 E2E 四路径全 PASS（假 VM 不破坏 vm-a3e86d）

## 笔记

- 选 `cloud-init.vendor-data` 而非 `user-data`：vendor-data 与 user-data 合并，不覆盖用户的 SSH keys
- 选 `chpasswd.users[]` 结构而非 `runcmd` shell：runcmd 会被 cloud-init.log 带 cmd tracing 泄密
- `cloud-init.instance-id` bump 是重跑触发器，不 bump 则 cloud-init 认为同一 instance 跳过模块
- 不轮询 cloud-init 完成状态（agent 不可用才走 offline，没有可靠探针）；admin 需自行确认或再跑 online
- offline 耗时 30-90s（stop + image-dep 启动时间 + cloud-init 阶段），前端应 spinner
