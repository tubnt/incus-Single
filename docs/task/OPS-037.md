# OPS-037 修复 BUG-06 size token min()/calc() 静默失效

- **status**: completed
- **completedAt**: 2026-05-03
- **priority**: P1
- **owner**: claude
- **createdAt**: 2026-05-03
- **scope**: 用户截图反馈 Cmd+K 弹窗严重变窄（~110px）—— 排查发现 OPS-032 BUG-06 引入的 5 个 `--size-sheet-{md,sm,lg}` / `--size-{toolbar,sidebar}-fluid` token 的值是 `min()` / `calc()` CSS function，**Tailwind v4 不会把这种 function 渲染为 width utility**，导致 `w-sheet-lg` 等 class 静默失效

## 影响范围

| 组件 | 之前 className | 实际渲染 | 现象 |
|---|---|---|---|
| `command-palette.tsx` Popup | `w-sheet-lg` | width:auto | Cmd+K 弹窗收缩到 ~110px |
| `invoice-detail-dialog.tsx` Popup | `w-sheet-md` | width:auto | 发票详情弹窗变窄 |
| `vm-peek-panel.tsx` aside | `w-sheet-sm` | width:auto | VM peek 抽屉变窄 |
| `app-sidebar.tsx` Accordion.Trigger | `w-sidebar-fluid` | width:auto | 侧边栏分组按钮收缩 |
| `batch-toolbar.tsx` 工具条 | `max-w-toolbar-fluid` | max-width:none | 全宽（不限制） |

## 修复

5 处统一改 `style={{ width: "var(--size-...)" }}` 直接绑 CSS var。token 仍合规（来自 @theme），不是 hex 字面量也不是 arbitrary value，符合 DESIGN.md 严格约束。
