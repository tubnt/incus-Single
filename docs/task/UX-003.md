## 调整后的 Findings (G1-G5)

### G1 用户端充值入口(需产品决策)

- **现状**:Dashboard 顶部显示余额,但 `/billing` 无充值入口;后端**没有** portal 侧 TopUp 路由;admin 端 `POST /admin/users/{id}/balance` 会拒绝「自己给自己充值」(`user.go:125-127 actorID == id → 400`),即使有 admin 身份也不能绕过
- **业务本质**:当前系统**没有支付网关**,余额完全由 admin 人工上调(PLAN-012 日额度 + 并发锁保护)。此为产品级决策,无法靠前端 UI 补齐
- **三种选择**(需用户挑一个):
  - **A** 隐藏自助充值(推荐 MVP):用户余额变成「只读」,页面加「充值请联系管理员」提示 + 提工单入口。零后端工作量
  - **B** 新增 `POST /portal/topup` 创建一笔 pending 订单(不扣款,不发票),后台 admin 审核后通过 Top-up 完成入账。工作量:0.5d 后端 + 0.5d 前端
  - **C** 对接第三方支付(支付宝/微信/Stripe),改动量极大,**明确不在本期**

### G2 订单取消(已实现,仅需修技术债)

- **UI 现状**:`billing.tsx:215-238` `OrderRow` 已为 `status === 'pending'` 渲染「取消」按钮,调用 `useCancelOrderMutation`,成功/失败均有 toast
- **后端现状**:`POST /portal/orders/{id}/cancel` 已注册(`portal.go:37`),`Cancel` handler 做 owner 检查 + `status == pending` 守卫
- **技术债**:`order.go:270-291` 的 `Cancel` 走**独立 UPDATE 而非事务 + FOR UPDATE**,与 `Pay`(已带 FOR UPDATE)存在竞态:Pay 持锁→读 pending→扣款→提交 → Cancel 无锁 UPDATE 把 status 改回 cancelled。结果:余额已扣 + 发票已产生 + 订单 = cancelled
- **本期范围**:包 Cancel 进事务,用 `SELECT status FROM orders WHERE id=? FOR UPDATE`(或加 `WHERE id=? AND status='pending'` 的条件性 UPDATE 判断影响行数),消除竞态
- **前端无工作量**

### G3 发票「详情」(走前端拼接)

- **现状**:发票列表展示 `id / order_id / amount / status / paid_at`,无「详情」按钮
- **Model 约束**:`Invoice` 结构(`models.go:123-133`)仅有金额字段,**无 line_items 表**,一张订单对应一张发票对应一个产品。所谓「详情」不过是把发票行所对应的 `order → product` 展开显示(配置、时长、创建时间)
- **本期范围**:前端弹窗聚合 `invoices[i] + orders[i] + products[i]`(数据已在页面内,无新请求);后端不动
- **PDF 导出**:BACKLOG

### G4 用户自行关闭工单(需新增后端路由)

- **现状**:`ticket.go` `PortalRoutes`(line 24-27)只有 list/create/get/reply,**没有任何状态变更路由**;只有 `AdminRoutes`(line 43)的 `PUT /admin/tickets/{id}/status`
- **后端需新增**:`POST /portal/tickets/{id}/close` — owner 检查 + `status != closed` 守卫(幂等) + 写 `status=closed, closed_at=now()`。一条回复消息可选(系统自动)
- **前端**:`tickets.tsx` 用户自己创建 + `status !== 'closed'` 时加「关闭」按钮 + confirm dialog + `useCloseTicketMutation` invalidate `ticketKeys.all`

### G5 Dashboard 快捷操作(链接目标纠正)

- **现状**:`app/routes/index.tsx:27-39` 仅 3 个 StatCard,无 CTA;user 侧导航需从左侧菜单进入
- **纠正目标**:
  - 「+ 新建云主机」→ `/billing`(产品卡购买 + Pay 是唯一创建入口,**无** `/vms/new` 路由)
  - 「+ 充值」→ 触发 G1 选定方案(A=提工单 / B=Top-up Dialog)
  - 「+ 提工单」→ `/tickets`(列表页顶部已有表单,**无** `/tickets/new` 路由)
- **仅在用户视角显示**:admin 视角 Dashboard 已有 `AdminSection`,快捷操作只在 user 视角渲染
- **空态优先**:当 `myVmCount === 0` 或 `openTickets === 0` 时,对应按钮可加「推荐」高亮
