# OPS-042 多维度 PickNode 调度（mem + cpu + disk + maintenance 过滤）

## 描述

当前 `cluster.Scheduler.PickNode` 仅看 mem free ratio 单维度，会把 VM 放到"内存最空但 CPU 已 100% / 磁盘快满 / 维护态"的节点。本任务改为加权多维度评分。

加权设计：
- 0.5 * mem_free_ratio（主导，启动硬约束）
- 0.3 * (1 - vm_cpu_density)（vm_density = sum(VM cpu limits) / cpu_total）
- 0.2 * disk_free_ratio（拉 ceph-pool 或本地 pool 状态）
- maintenance（`scheduler.instance=manual`）/ Evacuated 节点直接剔出候选池

不引入 prom 数据；所有指标直接来自 Incus REST API（`/1.0/resources` + `/1.0/storage-pools/<pool>/state?target=X`）。
