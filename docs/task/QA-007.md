## P1 详情（建议优先修）

### BUG-01 SPA fallback 太宽（产线已观测）

```bash
$ curl -fsS http://127.0.0.1:8080/assets/launch-XXXX.js | head -c 200
<!doctype html>
<html lang="zh-CN" class="dark">...

$ curl http://127.0.0.1:8080/locales/notexist/common.json -w 'HTTP=%{http_code} ct=%{content_type}\n'
HTTP=200 ct=text/html; charset=utf-8
```

后端 `internal/server/static.go` 把所有未匹配请求回退到 index.html，**包括 `/assets/*` `/locales/*`**。后果：

- 用户上次打开页面缓存了旧 chunk hash → 新版本部署 → 用户切路由 → 拉 `launch-RJR28jaH.js`（旧 hash） → 拿到 index.html → 浏览器尝试 parse 成 module 报 `Failed to fetch dynamically imported module` → 整页白屏
- i18n 加载未支持的语言（如未来加 fr）会拿 HTML 做 JSON.parse，报错

**修复**：static handler 对 `/assets/` 和 `/locales/` 前缀返 真实 404（不 fallback）。SPA fallback 仅对其它所有路径生效。约 10 行 Go 改动。

### BUG-02 mutation 无 onError 静默失败

| 位置 | 影响 |
|---|---|
| `features/snapshots/snapshot-panel.tsx:40-47` create | 创建快照失败用户看不到 |
| `features/snapshots/snapshot-panel.tsx:49-53` delete | 删除失败无 toast |
| `features/snapshots/snapshot-panel.tsx:55-59` restore | 回滚失败静默 |
| `app/routes/admin/floating-ips.tsx` attach | 绑 IP 失败无反馈 |
| `app/routes/admin/floating-ips.tsx` detach | 解绑失败无反馈 |
| `app/routes/admin/floating-ips.tsx` release | 释放失败无反馈 |
| `app/routes/admin/os-templates.tsx` toggle | 启/禁失败静默 |
| `app/routes/admin/tickets.tsx` status | 工单状态变更失败静默 |

**修复**：每个 mutation 加 `onError: (err) => toast.error(err.message ?? "操作失败")`。8 处 ×2 行。
