## Verification (production, after deploy)

- N1: html lang attribute now syncs with i18next; CSS font-family fallback chain
  includes PingFang SC / Microsoft YaHei UI / Noto Sans CJK SC. Real users on
  systems with CJK fonts now render correctly. Headless Chromium without CJK
  fonts still shows tofu — this is a system-font limitation, not an app bug.
- N2: Portal `/tickets` row now shows ▶/▼ chevron + linkified Subject; row
  click expands to inline TicketDetail with reply form (existing wired API).
- N3: monitoring / observability / node-join / products / nodes / users / etc.
  all key-driven via `t()`; locale files filled in en/zh.
- N5: audit log target — vm.create now reads `details.name` first ("vm vm-aa6862")
  and falls back to `#id` only when no human-readable field exists.
- N6: `/admin/$unknown` shows styled 404 with "Back to Clusters" link.
- N8: `/api/missing` → 404 JSON `{"error":"not found"}`;
       `/api/admin/orders/99999` → 404 JSON `{"error":"order not found"}`;
       `/api/admin/products/99999` → 404 JSON `{"error":"product not found"}`.
       MethodNotAllowed for `/api/*` returns 405 JSON `{"error":"method not allowed"}`.
- N11: validHost extracted to `features/nodes/host-validation.ts` with unit tests.
- N12: all admin table wrappers `overflow-hidden` → `overflow-x-auto`; tables
  scroll horizontally on narrow viewports instead of being clipped.
- N13: portal `/vms` username now derived from image string via
  `defaultUserForImage()` — debian/rocky/almalinux/centos/fedora/opensuse/arch/
  alpine/freebsd/ubuntu mapping.
- N15: audit log IP column strips `/32`/`/128` CIDR suffix.
