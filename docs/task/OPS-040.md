## 描述

当前 `vms.location` 已存在于 DB / API / admin 列表 / 用户卡，但散落分布、不易在运营视角"一眼定位"，且只能单台迁移。本任务把 VM 节点分布提升为运营一级信息，提供：

1. admin/nodes 顶部 **VM 分布概览**（每节点 VM 数 + mem 利用 + 维护态徽章）
2. admin/vms 列表对 location 列开排序 + 按节点筛选
3. VM 详情页 location 提到 header 显眼位置
4. **批量迁移**（admin/vms BatchToolbar 加"迁移到..." → NodePicker 选目标 → SSE 进度卡）
5. 不均衡建议面板：识别"某节点 mem 利用 > 集群均值 +20%"，给出"建议迁出 N 台到 X"，一键提交批量迁移
6. 用户端 VM 卡：把 node 标签从 caption 行提到 status pill 旁，明示"在哪台"

不做：live migration（需 Ceph block + QEMU live-patch，依赖独立 plan）；自动 rebalance daemon（仅人工触发）。
