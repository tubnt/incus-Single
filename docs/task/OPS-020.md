# OPS-020 前端 arbitrary value 全量替换为 @theme utility 直引

- **status**: completed
- **priority**: P3
- **owner**: claude
- **createdAt**: 2026-04-30
- **completedAt**: 2026-04-30
- **relatedPlan**: PLAN-022 / PLAN-024（DESIGN.md 合规收尾）

## 背景

DESIGN.md §"无 hex 字面量、无 arbitrary value" 与 pma-web 的双重纪律下，
前端不应直接使用 Tailwind arbitrary value (`shadow-[var(--shadow-X)]` 等)。
项目 `index.css` 的 `@theme inline` 已经把所有视觉 token 桥接成 Tailwind v4
namespace（`--shadow-*` / `--font-weight-*` / `--color-*` / `--radius-*`），
但历史代码（PLAN-022 起）多数仍以 `[var(--token)]` 字符串方式调用。

## 需要替换的写法

| 现状（违规） | 替换为（合规） | 出现位置数量（grep 估算） |
|---|---|---|
| `shadow-[var(--shadow-dialog)]` | `shadow-dialog` | 9 处（dialog/popover/dropdown/sheet/tooltip/select/alert-dialog/command-palette/admin/ha） |
| `shadow-[var(--shadow-floating)]` | `shadow-floating` | 2 处（batch-toolbar / vm-peek-panel） |
| `shadow-[var(--shadow-ring)]` | `shadow-ring` | 1 处（card） |
| `font-[510]` | `font-emphasis`（`@theme` 已暴露 `--font-weight-emphasis: 510`） | 多处（vm-peek-panel / 各路由 / sidebar） |
| `font-[590]` | `font-strong` | 多处 |
| `hover:bg-[color:var(--accent-hover)]` | 把 `--accent-hover` 通过 `@theme` 暴露成 `--color-accent-hover`，再写 `hover:bg-accent-hover` | 1 处（vm-peek-panel） |

## 不在范围

- viewport 计算 `max-w-[calc(100vw-2rem)]` / `w-[min(92vw,24rem)]` —— 这类是
  CSS 函数表达式，无法 token 化，按 DESIGN.md 注释也允许。
- grid 模板 `grid-cols-[auto_1fr]` —— 同理，不属于 hex / 颜色 / 字号 / 间距 / 圆角 / 阴影 token 范畴。

## 落地要点

- 在 `index.css @theme inline` 暴露 `--color-accent-hover: var(--accent-hover)`
  让 `bg-accent-hover` utility 可用
- Tailwind v4 自动从 `--shadow-*` / `--font-weight-*` token 生成对应 utility
  （`shadow-{name}` / `font-{name}`）
- 替换覆盖 ~60 个文件、180+ 处 arbitrary value，统一执行避免新旧写法共存
- viewport 计算（`max-w-[calc(100vw-2rem)]`、`w-[min(92vw,24rem)]`）和 grid
  模板（`grid-cols-[auto_1fr]`）按 DESIGN.md 注释允许，不在替换范围
