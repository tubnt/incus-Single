# SEC-002 portal API cloud-init 字段过滤（含 password）

- **status**: completed
- **completedAt**: 2026-04-30 (合并到 OPS-028)
- **priority**: P2
- **owner**: claude
- **createdAt**: 2026-04-30
- **plan**: [PLAN-030](../plan/PLAN-030.md)

## Description

OPS-027 测试时发现 `/api/admin/clusters/{name}/vms` 响应里 `config.user.cloud-init` 字段是明文 cloud-config YAML，含 `password: <hex>` 行——这是 VM 初始化阶段写入 root 密码的常见模板。

admin 视角问题不大（admin 本来就有 reset-password 权限），但：
1. portal/vms（用户视角）也走同样的 instance JSON pass-through，理论上同样泄露
2. response 体积大且含敏感字段，应在 server 层裁剪
3. OPS-022 加密了 vms.password DB 字段；但 Incus instance config 里依然明文

## Acceptance criteria

- portal/vms 与 admin/vms 响应中 `config.user.cloud-init` / `expanded_config.user.cloud-init` 字段被过滤或脱敏
- 过滤逻辑放 handler 层（不动 Incus client raw 透传）
- 不破坏 admin reset-password / rescue 等流程（这些走独立 API）
- 加 unit test 覆盖过滤逻辑

## 参考

- OPS-022 加密路径：`internal/auth/password_crypto.go`
- 当前 admin/vms handler：`internal/handler/portal/vm.go::ListClusterVMs`
- 用户视角 handler：搜 `portal.*VMs`
