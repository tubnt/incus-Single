- **scope**: 修 PLAN-051 / Session-2 漏过的 cloud-init 主线缺陷：linuxcontainers
  `*/cloud` 镜像不预装 openssh-server，但 `buildCloudInit` 没注入
  `packages:` 导致新建 VM 永远 SSH 连不上（vm-08f9d5 实测）；同时把 user
  journey 闭环（等 cloud-init 完成 + 验证 SSH ready + DonePanel 凭据明文 +
  一键复制 + 统一 root + cloud_init_template 字段接入 + firewall binding
  漂移修复）一次性补齐。详细分阶段见 PLAN-052。

## 上游证据

- 实测：node1/customers/vm-08f9d5（ubuntu/24.04/cloud）`cloud-init status=done`
  + ubuntu shadow hash 长度 73 字符（密码确实写入），但 `/usr/sbin/sshd`
  不存在、`ss -ltn` 无 22 监听。`dpkg -l | grep ssh` 只有 `openssh-client`。
- 根因：`internal/service/vm.go:881 buildCloudInit` 模板只输出
  `password / chpasswd / ssh_pwauth`，**无 `packages:`**。
  linuxcontainers `*/cloud` 是 332 MB minimal 镜像，设计上不带 sshd。
- 影响面：所有 9 个 Linux OS 模板（ubuntu/debian/rocky/almalinux/fedora/arch）
  source 均 `*/cloud`，全部中招。Windows 走独立 applyWindowsCloudInit 路径
  不受影响。
