# OPS-012 Reinstall 数据丢失保护：删除前探测镜像服务器

- **status**: completed
- **priority**: P2
- **owner**: claude
- **createdAt**: 2026-04-28 18:52
- **completedAt**: 2026-04-28 18:54
- **relatedBug**: OPS-008 Bug #8（vm-8f8912 因 reinstall delete 后镜像拉不到 → VM 数据永久销毁）

## 上下文

OPS-008 Bug #8 教训：Reinstall 流程是 `delete original instance → re-create from image`。如果 delete 之后 create 失败（最常见原因：上游镜像服务器挂掉 / 网络分区 / 镜像 alias 不存在），原 VM 已删，无法 rollback —— **用户数据永久销毁**。

OPS-008 笔记：

> **未补救机制**：如果 create 失败（如镜像不存在），VM 已删除无法回滚。建议加 dry-run pre-check：先 `images:` 验证目标 source 可拉取，再 delete

本任务实现 P2 防御层：**delete 前 HTTP HEAD 探测镜像服务器可达性**。

## 范围 vs 局限

✅ **挡住的场景**（生产最常见的"数据丢失"路径）：
- 上游镜像 CDN（如 `images.linuxcontainers.org`）整站宕机
- 主控到镜像服务器的网络分区
- 用户填错 server URL（域名解析失败）

❌ **不挡住的场景**（remaining 1%）：
- 服务器在线但具体镜像 alias 不存在（需要走 simplestreams 解析才能验证）
- 服务器在 probe 时在线，delete 后才挂掉（race window 有但很短）

更彻底的方案（不在本任务范围）：
- 删除前先 `incus image copy` 拉到本地缓存，create 复用缓存
- 删除前打 snapshot，create 失败时 restore from snapshot

## 修复

### `service/vm.go`

新增 `probeImageServer(ctx, serverURL)` 函数：

```go
var reinstallImageProbeClient = &http.Client{Timeout: 8 * time.Second}

func probeImageServer(ctx context.Context, serverURL string) error {
    if serverURL == "" {
        return nil
    }
    probeURL := strings.TrimRight(serverURL, "/") + "/streams/v1/index.json"
    req, _ := http.NewRequestWithContext(ctx, http.MethodHead, probeURL, nil)
    resp, err := reinstallImageProbeClient.Do(req)
    if err != nil {
        return fmt.Errorf("image server unreachable: %w", err)
    }
    defer resp.Body.Close()
    if resp.StatusCode >= 400 && resp.StatusCode != http.StatusMethodNotAllowed {
        return fmt.Errorf("image server returned %d", resp.StatusCode)
    }
    return nil
}
```

`Reinstall(ctx, params)` 在 `client.APIDelete` 之前调用：

```go
if err := probeImageServer(ctx, serverURL); err != nil {
    return nil, fmt.Errorf("镜像服务器 %s 不可达，已取消重装以保护原 VM 数据: %w", serverURL, err)
}
```

错误消息中文友好（链路与 Bug #14 / OPS-010 一致），用户能从 toast 直接读懂"重装失败 + 数据安全"。

### 新单测 `vm_test.go::TestProbeImageServer` (4 case)

- 空 URL 不报错（默认上游兜底）
- 200 OK → 可达
- 405 Method Not Allowed（部分 CDN 不允许 HEAD）→ 视为可达
- 503 Service Unavailable → 不可达
- 不存在的端口 `127.0.0.1:1` → 不可达
