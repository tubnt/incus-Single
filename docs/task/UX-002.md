## Summary

Split the single sidebar into two perspective-driven sidebars:

- **User perspective** (`/`, `/vms`, `/billing`, etc.): flat 7-item menu
- **Admin perspective** (`/admin/*`): collapsible two-level menu grouped into 5 categories

Add a top-of-sidebar button (admin-only) to switch between perspectives.

## Why

Production pain: admin users see 23 menu items in a flat list. The 16 admin items
are unordered — finding "orders" requires scanning the whole column. User requested
grouping by "orders & finance" + "ops" plus a cleaner perspective split.

## Acceptance

- On `/`, sidebar shows only user items (Dashboard + 6 user pages)
- On `/admin/*`, sidebar shows 5 collapsible groups; current-path's group auto-expands
- Admin users see a "进入管理后台" / "返回用户后台" switch button at the sidebar top
- Non-admin users never see admin entries or the switch button
- i18n complete for zh + en
- Browser verified in collapsed + expanded sidebar states, both themes
