## 笔记

- **为什么不用 Incus network-forward**：br-pub `ipv4.address=none` 没 NAT，network-forward（DNAT）不 apply
- **为什么不用 BGP / VRRP**：基建无 peering，一期不铺基础设施
- **真实 attach 流程**：管理面 hypervisor 侧关 ipv4 filtering；guest OS 侧 `ip addr add X.Y.Z.W/26 dev eth0` + `arping -U` 由操作者或 cloud-init 跑
- **Floating IP 从哪里拿**：admin 显式指定 IP，不自动从 /26 池取（该池是常规 VM 主 IP 源，避免冲突）；runbook 明确列出候选范围（如 .55-.59）
- **竞态守卫**：`UPDATE ... WHERE status='available'` + row count 判 0→409，不用额外锁
- **状态机一致性**：CHECK 约束 `(status='available' AND bound_vm_id IS NULL) OR (status='attached' AND bound_vm_id IS NOT NULL)`；DB 层面兜底应用层 bug
