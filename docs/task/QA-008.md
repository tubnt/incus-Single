## 发现汇总（11 条）

### P1（影响主功能 / 视觉破坏）

#### BUG-A1 · UX-005 回归 — dashboard "新建云主机" 仍指向 `/billing`
- **位置**：`web/src/app/routes/index.tsx:48` (command action) + `:105` (Link href)
- **现象**：仪表盘页面"新建云主机"快速操作按钮 link 是 `/billing`，但 UX-005 把用户创建入口改到 `/launch`。命令面板中的同名 action 也是错的目标。
- **修复**：两处 `to="/billing"` → `to="/launch"`

#### BUG-A9 · OPS-032 回归 — admin/users 页标题缺失 i18n key
- **位置**：`locales/{zh,en}/common.json` `admin.users.title` 不存在
- **现象**：`admin/users.tsx:153` 使用 `t("admin.users.title", { defaultValue: "Users" })`。OPS-032 BUG-10 修重复键时把唯一的也删掉了，现在 zh 用户页面标题渲染异常（fallback 经 i18next 路径解析为父级或 defaultValue）。
- **修复**：补回 zh `"title": "用户"` / en `"title": "Users"` 在 `admin.users` 命名空间内

### P2（体验缺陷，多处 i18n 漏译 + UX 小坑）

#### BUG-A2 · VM 详情 InfoCard 4 个标签硬编码英文
- **位置**：`web/src/app/routes/vm-detail.tsx:348` 附近
- **现象**：在 zh locale 看到 `IP / Username / Node / Created` 全英文（不走 t()）
- **修复**：用 `t("vm.ip" / "vm.username" / "vm.node" / "vm.created")`

#### BUG-A6 · billing 表头 i18n 漏译
- **位置**：`locales/zh/common.json`
  - `admin.amount` zh 值 = `"Amount"`（英文）
  - `invoice.order` zh 值 = `"Order"`（英文）
  - `invoice.amount` / `invoice.status` 不存在 → fallback 到英文 defaultValue
- **现象**：billing 页发票表头展示英文 "Amount / Status / Order"
- **修复**：`admin.amount: "金额"` / `invoice.order: "订单"` / 加 `invoice.amount: "金额"` / `invoice.status: "状态"`

#### BUG-A8 · admin/clusters 表头全英文
- **位置**：`web/src/app/routes/admin/clusters.tsx`
- **现象**：`Node / Status / CPU / Memory / Free % / Actions` 6 个 column header 全英文硬编码
- **修复**：走 i18n key

#### BUG-A10 · admin/ha 多处英文硬编码
- **位置**：`web/src/app/routes/admin/ha.tsx`
- **现象**：
  - 3 个 stat card 大写 label `HA STATUS / STORAGE / NODES`
  - 状态文字 `Enabled / Online / Fully operational`
  - 列头 `Node / Status / Message / Actions`
- **修复**：全部走 i18n。状态值 `Enabled/Online/Fully operational` 来自后端响应或前端 enum 映射，需要分别处理（后端 enum 改前端展示层映射）

### P3（卫生 / 一致性）

#### BUG-A3 · `snapshot.create` zh 文案不通顺
- **位置**：`locales/zh/common.json:872` `snapshot.create: "快照"`
- **现象**：「快照」面板上的"创建"按钮 label 显示为"快照"——和面板标题重复且不像动词
- **修复**：改为 `"新建快照"`

#### BUG-A7 · 直接访问 `/admin` 跳到 `/`
- **位置**：路由 `/admin` 没 index 子路由
- **现象**：用户输入 `/admin` 期望去 admin 默认页，实际被跳到根（用户视角的 dashboard）
- **修复**：在 `routes/admin.tsx` 加 `beforeLoad` 或 `loader` redirect 到 `/admin/monitoring`，或者新建 `routes/admin/index.tsx` 作 redirect

#### BUG-A11 · Mobile launch summary card 与 fixed bar 视觉挤迫
- **位置**：`web/src/app/routes/launch.tsx` mobile 布局
- **现象**：375px 视口下 summary card 出现在表单段落下方（不再 sticky），底部有 fixed mobile bottom bar；scroll 到底部 summary 紧贴 bar 上边缘
- **修复**：mobile 下 summary card 隐藏（`md:block hidden` 或类似），主 CTA 走 mobile bottom bar；或者 summary 留但 padding 加大
