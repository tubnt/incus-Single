## 描述

把硬编码在 `web/src/features/vms/os-image-picker.tsx` 的 4 条 OS 镜像迁到 DB，支持：
- 后端 `os_templates` 表（slug / name / source / protocol / default_user / cloud_init_template / enabled / sort_order / supports_rescue 预留）
- admin CRUD（/admin/templates）
- portal 只读（/api/templates）
- Reinstall 流程改走 template_slug

### 范围

**Phase A 镜像模板 DB 化（3d）**：
- migration 009 `os_templates` + seed（Ubuntu 24/22/20、Debian 12/11、Rocky 9、Alma 9、Arch、Fedora 40）
- `internal/repository/os_template.go` CRUD
- `internal/handler/portal/template.go` GET /api/templates + admin 写接口
- 前端 features/templates/api.ts + 重写 os-image-picker.tsx 从 DB 读
- admin /admin/templates 页面

**Phase B Reinstall 走模板 + UI 接入（2d）**：
- `VMService.Reinstall` 改签：`ReinstallParams{TemplateSlug string}` 替代 `NewOSImage`
- 前端 VM 详情页"重装系统"dialog
