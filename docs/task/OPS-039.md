## 描述

当前 `/admin/node-join` 让运维填 12 个字段（私钥路径、6 个 IP、3 个网卡名、role 等），且私钥必须预先扔到 admin 服务器固定路径，门槛高、误操作风险大。

参考 Proxmox/Rancher/MAAS 业界做法，本任务要把"添加节点"这一动作改造为：
1. **凭据三选一**：密码 / 粘贴私钥内容 / 已部署的 SSH key 文件（兼容老路径）。
2. **自动探测**：admin 服务器一次性 SSH 进目标机，采集 hostname / OS / CPU / 内存 / 网卡列表（含 bond）/ 已配 IP / 默认路由 / 公网 IP / 磁盘，回填表单。
3. **UI 由"输入"变"确认"**：表单只剩"凭据 + 角色"两块必填，其余字段以探测结果预填，运维只在异常时才覆盖。
4. **现有 9 步执行链不动**（`leader_token → ... → verify`），仅在前端入口加一个 wizard 收敛流程。

集群创建（PLAN-027 留下的纯 SQL upsert）不在本任务范围，后续单独立项。

## 验收标准

- 新建节点时只需填：① 主机/端口 ② 凭据（密码或私钥）③ 角色，提交"探测"按钮 5 秒内拿到节点信息确认页。
- 旧的 `ssh_key_file` 路径模式继续可用（向后兼容）。
- 所有凭据落 DB（用 OPS-022 既有 AES-256-GCM）；密码模式默认仅用一次性。
- 在 vmc.5ok.co 真机跑通 node1（已有 5G NIC，非 bond）和 node6（bond 拓扑）两类节点的探测+加入。
- go vet / go test / tsc / vitest / build 全绿；DESIGN.md token 严格合规（无 hex / arbitrary value）。
